# docs/ — design narrative

> These files are **thin generic placeholders**. They describe the *shape*
> the boilerplate ships with. As your project grows, replace the generic
> text with the exact architecture, tables, and flows of your service — in
> the same commit as the code change (see the documentation rule in the
> root [`CLAUDE.md`](../CLAUDE.md)).

| File | Covers |
|---|---|
| [`architecture.md`](architecture.md) | Layered structure, request lifecycle, resilience layer |
| [`erd.md`](erd.md) | Database tables (entity-relationship diagram) |
| [`class-diagrams.md`](class-diagrams.md) | Base classes + exception hierarchy |
| [`security.md`](security.md) | Security headers, CORS, rate limits, audit log |

Diagrams use [Mermaid](https://mermaid.js.org/), which GitHub renders inline.
